SECTION — FUTURE STRATEGY

AX (AI Transformation)

청주김안과에 도입 제안드리는 AI 전환 로드맵 — 자동화, 예측, 지능화를 통한 진료·상담·운영 효율 극대화

전제: AX는 도입 로드맵의 전면 오픈(W12) 이후에 착수하는 여정입니다. 본 페이지의 Phase 1~3 기간은 오픈 이후를 기준으로 하며, 구체 일정·투자 규모는 파일럿 결과와 청주김안과 규모에 맞춰 별도 산정합니다.

2. AX 3단계 로드맵

Phase 1 — Foundation
오픈 후 3 ~ 6 개월 · 데이터 기반 자동화
  • 중앙 관측 체계 확장 (로그/성능/에러 집중 수집)
  • DB 성능 인덱스 PROD 전면 적용
  • MSSQL → PostgreSQL 점진 이관 검토 (오픈소스 전환)
  • 고객/상담/리콜/수술 이력 데이터 정제 + 라벨링
  • 규칙 기반 이상 감지 (재고·예약 충돌·수납 불일치)
핵심: AI 학습을 위한 데이터 인프라 확보
Phase 2 — Intelligence
오픈 후 6 ~ 12 개월 · AI 예측 + 추천
  • 재내원 확률 예측 모델 (리콜 우선순위)
  • 소개 고객 자동 매칭 (Embedding 유사도)
  • 상담 메모 요약 + 키워드 태깅 (LLM)
  • 수술 수요 예측 + 스케줄 제안
  • SMS 문구 개인화 + 발송 시점 최적화

※ 예측 모델 도입 시점은 학습에 필요한 데이터 축적 수준에 따라 조정합니다.

핵심: 데이터 기반 의사결정 지원
Phase 3 — Autonomy
오픈 후 12 ~ 18 개월 · 자율 운영 지원
  • 상담사 AI 어시스턴트 (상담 중 참고 정보 실시간 추천)
  • 자연어 통계 질의 보조 (예: 월별 수술 건수·전환율 요약)
  • 진료 차트 작성 보조 — 의사의 검토·승인 전제
  • 환자 여정 자동화 (예약→내원→수술→AS 리콜 흐름 연결)
핵심: 사람의 판단을 AI가 옆에서 보조

3. 핵심 AI 활용 영역

★ 1. 지능형 리콜 대상 예측

현재 문제: 리콜 명단을 기간 필터와 상담사 감으로 선정하다 보니, 재내원 가능성이 높은 고객과 낮은 고객이 섞여 상담 시간이 낭비됩니다.

AI 적용 방안
  • 학습 데이터: 과거 리콜 → 실제 재내원 여부, 수술 이력, 상담 이력, 나이/지역
  • 모델: Gradient Boosting 등 예측 모델로 재내원 확률 산출
  • 동작: 리콜 리스트를 확률 상위부터 자동 정렬
  • 피드백 루프: 실제 내원 결과를 재학습에 반영
기대 효과: 리콜 재내원율 상승, 상담사 업무 시간 경감 (파일럿에서 정량 측정)
◈ 2. 소개 고객 자동 매칭

현재 문제: 소개자 코드를 상담사가 수동 입력하다 보니 이름·전화 오타, 동명이인 등으로 매칭 실패가 잦아 소개 보상·통계가 왜곡됩니다.

AI 적용 방안
  • 이름 + 전화번호 + 생년월일 임베딩 기반 유사도 검색
  • 높은 신뢰도 구간에서만 자동 매칭, 그 미만은 후보 상위 3건 추천
  • 상담사의 수정 이력을 학습 데이터로 반영
기대 효과: 오매칭 대폭 감소, 소개 보상 정합성 확보
⚠ 3. AI 기반 이상 감지 + 알림

아래 항목은 AI가 "정상 패턴을 학습한 뒤 평소와 다른 흐름"을 감지하는 영역입니다. 단순 규칙으로 차단하는 것(예: 예약 이중 배정, 수납 중복)은 동시성 / 데이터 정합성 페이지의 DB 제약·Idempotency Key로 처리합니다.

감지 대상방법대응
재고(렌즈/약품) 소진소진 속도 예측선제 발주 알림
개인정보 과다 조회평소 패턴 대비 조회량 이상치감사 로그 + 관리자 알림
상담 전환율 급락시계열 이상치 탐지원인 분석 리포트
수술 취소/노쇼 급증시즌 대비 편차사유 분류 + 개선 제안
Phase 1: 규칙 기반 기본 감지 → Phase 2: 시계열·ML 기반 확장
✎ 4. LLM 기반 운영 지원 (AI Agent)
4-1. 상담사 어시스턴트 — 기능 상세
기능설명병원 관점 가치
실시간 상담 분석상담 음성을 STT로 변환해 핵심 키워드·환자 니즈를 실시간 파악상담 내용의 데이터화 · 검색 가능
멀티 AI 에이전트상담 어드바이스 + 설명 보조 동시 지원상담사 숙련도와 무관한 일관된 안내 품질
고객 이력 카드초진/재진 고객별 과거 상담·수술 이력 자동 요약재방문 시 맥락 유지, 상담사 교체에도 끊김 없는 상담
다국어 통역실시간 AI 통역으로 외국인 환자 상담 지원통역 인력 부재로 인한 외국인 환자 이탈 방지
상담 품질 평가상담 종료 후 태도·공감도·설명력을 객관 지표로 평가상담사별 성과 수치 비교 · 교육 근거 확보
4-1-a. 핵심 가치
  • 상담 표준화 — 베테랑 노하우 전 직원 공유
  • 성과 가시화 — 품질 점수 실시간 비교
  • 인력 리스크 해소 — 퇴사에도 맥락 유지
4-2. 운영 Agent (Ops Agent)
  • "오늘 미수납 환자 보여줘" → DB 조회 + 요약
  • "이번 주 수술 대비 렌즈 재고 괜찮아?"
  • "라식 광고 링크 클릭 → 내원 전환율" 자연어 통계
4-3. 개발 지원 Agent
  • 코드 리뷰 자동화 (개발 표준 규칙 위반 검출)
  • 회귀 테스트 생성 지원
  • T-SQL 프로시저 → 서비스 레이어 마이그레이션 지원
기술 기반: Claude API + Agent SDK + MCP (DB 읽기 전용 권한 분리)
⚙ 5. 수술/예약 수요 예측

현재 문제: 일정 조율이 사람 감각에 의존해 요일별 편차가 발생할 수 있습니다. 인력·장비가 특정 요일에 몰리거나 비는 문제를 사전에 보이지 않습니다.

AI 적용 방안
  • 시계열 예측 기반 시술별 주간/월간 수요 추정
  • 계절 · 시즌 요인 반영 (연말, 방학, 기념일 등)
  • 인력/장비/수술방 배치 최적화 제안 (최종 결정은 실장·원장)

4. 참고: 기술 기반 구성

AX를 실제로 구동하는 기술 요소를 비즈니스 용어로 요약했습니다. 구체 도구 선택은 도입 시점에 최적 구성으로 확정합니다.

AI / 예측
  • 대형 언어 모델 (LLM) API — 요약·태깅·대화
  • 시계열 예측 엔진 — 수요·재고·추세
  • 분류·회귀 모델 — 재내원 확률 예측
  • 임베딩 기반 유사도 검색 — 소개 매칭
Data
  • MSSQL (현재 운영)
  • PostgreSQL (이관 검토)
  • 세션 · 캐시 저장소
  • 오브젝트 스토리지 (차트/서명 이미지)
Ops
  • 구조화 로깅 (운영 중)
  • 중앙 관측 플랫폼 (검토)
  • 컨테이너 배포 (중장기)

※ "검토"는 청주김안과 규모·요구에 따라 도입 여부·시점을 별도 판단합니다.

7. 결론

비쥬웍스가 아이차트 위에서 제공하는 AX 전략은 축적된 데이터를 실제 업무 효율로 연결하는 데 초점을 둡니다.

  • 단기: 관측성 + DB 성능 + PostgreSQL 전환 검토로 AI 학습 기반 확보
  • 중기: 리콜 예측, 소개 매칭, 상담 요약으로 운영 자동화 핵심 완성
  • 장기: 상담사/운영 AI Agent로 자율 운영 전환
AX는 하루에 오지 않습니다. 전면 오픈 이후의 운영 데이터가 쌓이는 만큼, 예측·자동화의 정확도도 함께 성장합니다.
저희는 그 데이터가 쌓이는 과정을 함께 설계·운영하는 파트너가 되겠습니다.

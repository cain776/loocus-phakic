SECTION — TECHNICAL FOUNDATION

기술적 기반 (Foundation)

청주김안과에 도입될 시스템은 표준화된 아키텍처, 검증된 DB 성능, 체계적인 관측 체계 위에서 운영되고 있습니다.

✦ 제안드리는 시스템은 AX(AI Transformation)를 즉시 시작할 수 있는 수준의 엔지니어링 기반을 이미 확보하고 있습니다.
코드 표준화, 성능 측정 문화, 보안 미들웨어 체계 — 이 세 가지는 하루아침에 만들 수 없는 자산이며, 수년간의 운영 경험을 통해 축적해 온 결과물입니다.

◉
표준화된 아키텍처
Route / Service / Schema 3레이어 패턴을 전사 표준으로 정착. 31개 서비스 모듈이 일관된 규칙으로 분리되어 있습니다.
⚡
실측된 DB 성능
커버링 인덱스 적용으로 핵심 쿼리 p95를 최대 89% 단축. 모든 성능 개선은 k6 기반 정량 측정으로 검증합니다.
🔍
요청 단위 추적 체계
requestLogger + reqId 컨텍스트로 모든 요청을 사용자·세션 단위로 추적. 장애 원인을 즉시 파악할 수 있습니다.

1. 표준화된 아키텍처

◉ 일관된 3레이어 패턴

모든 신규 기능은 Route → Service → Schema 3레이어 패턴으로 개발됩니다. 이 규칙이 CLAUDE.md에 명문화되어 있어, 어떤 개발자가 합류해도 동일한 구조로 코드를 작성합니다.

routes/{name}.js → services/{name}-service.js → schemas/{name}.schema.js HTTP 계층 비즈니스 로직 입력 검증(Zod)
현재 규모
  • 라우트 82개 (routes/*.js) — HTTP 엔드포인트
  • Service 레이어 31개 (services/*.js) — 비즈니스 로직 캡슐화
  • Zod 스키마 77개 (schemas/*.js) — 입력 검증 자동화
◉ 도메인별 서비스 분리

복잡한 도메인은 단일 서비스가 아니라 역할별 세부 모듈로 분리되어 있습니다. 이 구조 덕분에 향후 MSA 전환 시 프로세스 경계로 그대로 승격할 수 있습니다.

인증 도메인
  • auth-verify.js — 인증 로직
  • password-store.js — 비밀번호 관리
Agent Office
  • agent-office-service.js — 메인 로직
  • agent-office-data.js — 데이터 액세스
  • agent-office-emitter.js — 이벤트 방출
Customer 도메인 (7개 서비스)
  • customer-read / customer-write / customer-response / customer-response-write
  • customer-etc-panels / customer-sms / customer-visit-motive
◉ 체계적인 미들웨어 파이프라인

모든 요청은 보안 → 세션 → 로깅 → 인증 → 라우팅 → 에러 처리의 일관된 파이프라인을 통과합니다. 이 체계는 OWASP 권고안을 기반으로 설계되었습니다.

helmet → cors → json → responseHandler → sessionConfig → requestLogger → 정적파일 → limiter → csrf → requireAuth → 라우트 → globalErrorHandler
✔ 모든 보안 헤더, CSRF, Rate Limit, 세션 검증이 라우트 작성 없이 자동 적용됩니다.

2. 검증된 DB 성능

⚡ 실측 기반 성능 개선

제안 시스템의 DB 성능 개선은 "감"이 아닌 "측정"으로 진행됩니다. 모든 인덱스 적용은 k6 시나리오와 smoke 테스트로 전후 p95 latency를 비교 검증합니다.

WORK DB 커버링 인덱스 성과
인덱스대상 쿼리개선 결과 (p95)
IX_CUSTOM_CustName_Covering고객 이름 검색1.75s → 465ms (-73%)
IX_MOTIVE_NEW02_fkey_Idx예약 리콜7.14s → 764ms (-89%)
IX_CUSTOM_HISTORY_CustNum고객 이력 조회3.3s → 0.1s (-97%)
IX_MsgSndLst_Phone_SendDateTimeSMS 이력 조회 (대량 테이블)7s → 0.5s (-93%)
SP_CUSTOM_MEMO 프로시저 분해메모 조회14s → 0.9s (-94%)
✔ 상담사와 데스크가 체감하는 "느렸던 화면들"이 평균 80% 이상 빨라진 실적을 확보했습니다.
⚡ 검증 자동화 도구

성능 변경이 일회성에 그치지 않도록, 지속적으로 측정할 수 있는 도구 체계를 갖추고 있습니다.

k6 부하 시나리오
  • 시나리오 03 ~ 06 — 고객 검색, 리콜 등 핵심 플로우
  • p50/p95/p99 latency 자동 측정
Smoke 테스트 5종
  • auth / login-history
  • customer-all-panels / customer-update
  • prod-compare — WORK/PROD 응답 비교
⚡ 멀티 DB 통합 운영

제안 시스템은 SOFTCRM · MCRM · IPLUS 세 개의 DB를 단일 커넥션 풀로 통합 운영합니다. 크로스 DB 쿼리와 Linked Server 연동을 통해 분산 데이터를 매끄럽게 조회합니다.

  • 메인 테이블 CUSTOM — 40+ 컬럼, 고객 마스터
  • MsgSndLst — 대량 SMS 이력 테이블 (인덱스 최적화 완료)
  • CUSTOM_MEMO — RTF 기반 상담 메모
  • JUMIN_NUM — DECRYPTBYPASSPHRASE 기반 암호화 저장 (개인정보보호법 준수)

※ 위 테이블 구조·규모는 비쥬웍스가 운영 중인 S-CRM 계열 DB의 사례이며, 청주김안과의 실제 데이터 규모는 Phase 1 현황 분석에서 확인합니다.

3. 요청 단위 추적 체계

🔍 구조화된 로깅

모든 요청은 reqId, userId, 세션 컨텍스트가 포함된 구조화된 로그를 자동 생성합니다. 어떤 사용자가 언제 무엇을 했는지 즉시 추적할 수 있습니다.

winston 기반 로거
  • 카테고리별 로그 레벨 분리
  • 에러 스택 트레이스 자동 수집
  • 파일 로테이션 지원
requestLogger 미들웨어
  • 요청별 reqId 자동 발급
  • req.logger 컨텍스트 주입
  • 사용자 · 세션 정보 자동 태깅
🔍 중앙 집중 에러 처리

globalErrorHandler를 통해 모든 예외가 한 곳에서 포착되고 분류됩니다. 개발자는 라우트마다 try/catch를 작성할 필요가 없으며, 에러 응답 포맷은 전사 표준으로 통일되어 있습니다.

✔ 에러 발생 시 reqId 하나로 사용자 요청 → 서비스 호출 → DB 쿼리까지 역추적 가능
🔍 CLAUDE.md 기반 개발 표준

모든 로깅·에러 처리 규칙이 CLAUDE.md에 문서화되어 있어, 신규 개발자도 첫 PR부터 표준을 따를 수 있습니다. 이 문서는 AI 개발 어시스턴트(Claude Code)도 동일하게 참조합니다.

★ 왜 이것이 중요한가

많은 의료 시스템이 "돌아가기만 하면 된다"는 수준에 머무릅니다. 저희가 제안드리는 시스템은 그 이상을 추구합니다:

  • 표준화된 코드는 AI 도구가 읽고 수정할 수 있는 기반이 됩니다.
  • 측정된 성능은 개선이 실제 개선인지 증명할 수 있게 합니다.
  • 추적 가능한 요청은 장애가 아닌 "학습 데이터"가 됩니다.
이 세 가지가 갖춰져 있기에, 청주김안과는 도입 즉시 AX(AI Transformation)의 출발선에 설 수 있습니다.
AI 도입은 도구의 문제가 아니라 기반의 문제이며, 저희는 그 기반을 이미 준비해 두었습니다.
표준화된 코드 · 측정된 성능 · 추적 가능한 요청 — 비쥬웍스는 이 세 가지로 시작합니다
